## 1. 项目结构

```
src/
├── api/                    # 空目录，预留 API 服务层
├── assets/
│   └── main.css            # Tailwind v4 主题配置、全局样式
├── components/
│   ├── charts/             # 图表组件
│   ├── common/             # 通用组件（面包屑、卡片）
│   ├── ecommerce/          # 仪表盘业务组件
│   ├── forms/              # 表单组件
│   ├── icons/              # 自定义 SVG 图标组件（50+）
│   ├── layout/             # 布局组件（AdminLayout、Sidebar、Header）
│   ├── profile/            # 个人页组件
│   ├── tables/             # 表格组件
│   └── ui/                 # UI 原子组件（Button、Badge、Modal、Avatar、Alert）
├── composables/
│   └── useSidebar.ts       # Sidebar 状态管理组合式函数
├── router/
│   └── index.ts            # 路由定义
├── views/
│   ├── Auth/               # 登录/注册
│   ├── Chart/              # 图表展示页
│   ├── Errors/             # 404 页面
│   ├── Forms/              # 表单展示页
│   ├── Others/             # 日历、用户资料
│   ├── Pages/              # 空白页
│   ├── Tables/             # 表格展示页
│   ├── UiElements/         # UI 元素展示页
│   └── Ecommerce.vue       # 仪表盘首页
├── main.ts                 # 应用入口
└── App.vue                 # 根组件（ThemeProvider + SidebarProvider 包裹）
```

**注意：** 当前项目中没有 `stores/`、`types/`、`utils/`、`services/` 目录，如需使用请自行创建。

## 2. 新建页面完整流程

### 步骤 1：创建视图文件

在 `src/views/<FeatureGroup>/<PageName>.vue` 中创建页面，使用 **PascalCase** 命名：

```vue
<!-- src/views/Orders/OrderList.vue -->
<script setup lang="ts">
import { ref } from 'vue'
import AdminLayout from '@/components/layout/AdminLayout.vue'
import PageBreadcrumb from '@/components/common/PageBreadcrumb.vue'

const title = ref('订单管理')
</script>

<template>
  <AdminLayout>
    <PageBreadcrumb
      :items="[
        { title: '首页', href: '/' },
        { title: '订单管理' }
      ]"
    />
    <div class="grid grid-cols-12 gap-4 md:gap-6">
      <!-- 页面内容 -->
    </div>
  </AdminLayout>
</template>
```

### 步骤 2：注册路由

在 `src/router/index.ts` 中添加路由，使用 **懒加载** 并设置 `meta.title`：

```ts
{
  path: '/orders',
  name: 'OrderList',
  component: () => import('../views/Orders/OrderList.vue'),
  meta: {
    title: '订单管理',
  },
}
```

### 步骤 3：选择布局

- **仪表盘页面**：使用 `<AdminLayout>`（包含侧边栏和顶部导航）
- **全屏页面**（登录、注册等）：使用 `<FullScreenLayout>`

### 步骤 4：编写样式

使用 Tailwind CSS v4 工具类，**必须** 提供 `dark:` 暗黑模式变体：

```html
<div class="bg-white dark:bg-gray-900 text-gray-800 dark:text-white/90">
```

## 3. 路由规范

| 项目 | 规范 |
|------|------|
| 配置文件 | `src/router/index.ts` |
| 加载方式 | 全部使用懒加载 `() => import('../views/...')` |
| 路由结构 | 扁平结构，**不使用**嵌套路由 |
| 路径命名 | 短横线命名（kebab-case），如 `/form-elements`、`/basic-tables` |
| 页面标题 | 每个路由必须定义 `meta.title`，通过 `beforeEach` 自动更新 document.title |
| 视图目录 | `src/views/<FeatureGroup>/<PageName>.vue` |

路由标题更新逻辑：

```ts
router.beforeEach((to, from, next) => {
  document.title = `Vue.js ${to.meta.title} | TailAdmin - Vue.js Tailwind CSS Dashboard Template`
  next()
})
```

## 4. 组件规范

### 命名规范

| 场景 | 命名方式 |
|------|----------|
| Vue 文件 | PascalCase，如 `EcommerceMetrics.vue`、`DefaultInputs.vue` |
| 模板中使用 | 短横线命名（kebab-case），如 `<ecommerce-metrics />`、`<admin-layout>` |
| 目录名 | 功能目录使用 PascalCase，如 `FormElements/`、`UiElements/` |

### 组件组织

| 目录 | 用途 | 示例 |
|------|------|------|
| `components/ui/` | UI 原子组件，高度可复用 | Button、Badge、Modal、Avatar |
| `components/common/` | 通用组件 | PageBreadcrumb、ComponentCard、ThemeToggler |
| `components/layout/` | 布局组件 | AdminLayout、AppSidebar、AppHeader |
| `components/<feature>/` | 业务专属组件 | ecommerce/、charts/、tables/ |

### Props 定义规范

必须使用 `<script setup lang="ts">` + `interface` + `withDefaults`：

```ts
<script setup lang="ts">
interface ButtonProps {
  size?: 'sm' | 'md'
  variant?: 'primary' | 'outline'
  startIcon?: object
  disabled?: boolean
  className?: string
}

const props = withDefaults(defineProps<ButtonProps>(), {
  size: 'md',
  variant: 'primary',
  className: '',
  disabled: false,
})
</script>
```

## 5. 样式规范（Tailwind CSS v4）

本项目 **没有** `tailwind.config.js`，所有配置在 `src/assets/main.css` 中：

```css
@import 'tailwindcss';

@custom-variant dark (&:is(.dark *));

@theme {
  --font-outfit: Outfit, sans-serif;
  --breakpoint-2xsm: 375px;
  --breakpoint-xsm: 425px;
  --breakpoint-3xl: 2000px;

  --text-title-sm: 30px;
  --text-title-sm--line-height: 38px;
  --text-theme-sm: 14px;
  --text-theme-sm--line-height: 20px;

  --color-brand-500: #465fff;
  --color-success-500: #12b76a;
  --color-error-500: #f04438;
  --color-warning-500: #f79009;
  --color-gray-800: #1d2939;
  --color-gray-900: #101828;

  --shadow-theme-xs: 0px 1px 2px 0px rgba(16, 24, 40, 0.05);
  --shadow-theme-sm: 0px 1px 3px 0px rgba(16, 24, 40, 0.1), 0px 1px 2px 0px rgba(16, 24, 40, 0.06);

  --z-index-99999: 99999;
}
```

### 自定义工具类（@utility）

```css
@utility menu-item {
  @apply relative flex items-center w-full gap-3 px-3 py-2 font-medium rounded-lg text-theme-sm;
}

@utility menu-item-active {
  @apply bg-brand-50 text-brand-500 dark:bg-brand-500/[0.12] dark:text-brand-400;
}

@utility no-scrollbar {
  &::-webkit-scrollbar { display: none; }
  -ms-overflow-style: none;
  scrollbar-width: none;

```

### 暗黑模式规范

- 暗黑模式通过 `document.documentElement` 上的 `dark` 类触发
- **每个有样式的元素都必须写 `dark:` 变体**
- 常用模式：
  ```html
  class="bg-white text-gray-800 dark:bg-gray-900 dark:text-white/90 dark:border-gray-800"
  ```

### 输入框样式标准

所有输入框使用统一的类名字符串：

```html
<input
  class="dark:bg-dark-900 h-11 w-full rounded-lg border border-gray-300 bg-transparent px-4 py-2.5 text-sm text-gray-800 shadow-theme-xs placeholder:text-gray-400 focus:border-brand-300 focus:outline-hidden focus:ring-3 focus:ring-brand-500/10 dark:border-gray-700 dark:bg-gray-900 dark:text-white/90 dark:placeholder:text-white/30 dark:focus:border-brand-800"
```

## 6. 状态管理

本项目 **没有** 使用 Pinia 或 Vuex，状态管理通过 Vue 的 `Provide/Inject` + Composables 实现。

### 现有模式

**主题状态**（`ThemeProvider.vue`）：

```vue
<script setup>
import { provide, ref } from 'vue'

const isDarkMode = ref(false)
const toggleTheme = () => {
  isDarkMode.value = !isDarkMode.value
  document.documentElement.classList.toggle('dark')
}

provide('theme', { isDarkMode, toggleTheme })
</script>
```

**侧边栏状态**（`useSidebar.ts`）：

```ts
const SidebarSymbol = Symbol('sidebar')

export function useSidebar() {
  const context = inject(SidebarSymbol)
  if (!context) throw new Error('useSidebar must be used within SidebarProvider')
  return context
}
```

**App.vue** 包裹整个应用：

```vue
<template>
  <ThemeProvider>
    <SidebarProvider>
      <RouterView />
    </SidebarProvider>
  </ThemeProvider>
</template>
```

### 新增全局状态规范

如需新增全局状态，请遵循以下模式：

1. 创建 `*Provider.vue` 组件，调用 `provide(...)`
2. 创建 `use*()` 组合式函数，调用 `inject(...)`
3. 在 `App.vue` 中包裹新 Provider

## 7. TypeScript 规范

- 组件必须使用 `<script setup lang="ts">`
- `tsconfig.json` 启用 `strict: true`、`noUnusedLocals: true`、`noUnusedParameters: true`
- 路径别名 `@/*` 映射到 `src/*`
- 没有集中的 `types/` 目录，类型通常内联在组件文件中

### Props 接口命名

接口名以 `Props` 结尾，如 `ButtonProps`、`BadgeProps`：

```ts
type BadgeVariant = 'light' | 'solid'
type BadgeSize = 'sm' | 'md'
type BadgeColor = 'primary' | 'success' | 'error' | 'warning' | 'info' | 'light' | 'dark'

interface BadgeProps {
  variant?: BadgeVariant
  size?: BadgeSize
  color?: BadgeColor
}

const props = withDefaults(defineProps<BadgeProps>(), {
  variant: 'light',
  color: 'primary',
  size: 'md',
})
```

### 模块声明

第三方库的类型声明在 `src/index.d.ts` 中：

```ts
declare module 'jsvectormap'
declare module 'swiper/css'
declare module 'swiper/css/navigation'
declare module 'swiper/css/pagination'
```

## 8. 表单规范

- 使用原生 HTML 元素（`<input>`、`<select>`、`<textarea>`）+ `v-model`
- **没有** 验证库（VeeValidate、Zod 等），如需请自行引入
- 日期选择器使用 `vue-flatpickr-component`

### 标准表单模式

```vue
<script setup lang="ts">
import { ref } from 'vue'

const email = ref('')
const password = ref('')
const keepLoggedIn = ref(false)

const handleSubmit = () => {
  console.log('Form submitted', { email: email.value, password: password.value })
}
</script>

<template>
  <form @submit.prevent="handleSubmit">
    <div>
      <label class="mb-1.5 block text-sm font-medium text-gray-700 dark:text-gray-400">
        邮箱 <span class="text-error-500">*</span>
      </label>
      <input
        v-model="email"
        type="email"
        placeholder="info@gmail.com"
        class="dark:bg-dark-900 h-11 w-full rounded-lg border border-gray-300 bg-transparent px-4 py-2.5 text-sm text-gray-800 shadow-theme-xs placeholder:text-gray-400 focus:border-brand-300 focus:outline-hidden focus:ring-3 focus:ring-brand-500/10 dark:border-gray-700 dark:bg-gray-900 dark:text-white/90 dark:placeholder:text-white/30 dark:focus:border-brand-800"
      />
    </div>
  </form>
</template>
```

## 11. 新功能开发 checklist

开发新功能或页面时，请逐项检查：

- [ ] **创建视图文件** → `src/views/<FeatureGroup>/<PageName>.vue`
- [ ] **注册路由** → `src/router/index.ts`，懒加载 + `meta.title`
- [ ] **选择布局** → `<AdminLayout>` 仪表盘页面，`<FullScreenLayout>` 全屏页面
- [ ] **创建业务组件** → `src/components/<feature>/`
- [ ] **使用 UI 原子组件** → 从 `src/components/ui/` 导入（Button、Badge、Modal）
- [ ] **使用通用组件** → `PageBreadcrumb`、`ComponentCard`
- [ ] **应用 Tailwind 样式** → 使用主题色（`brand-500`、`success-500` 等）+ `dark:` 变体
- [ ] **使用图标** → 从 `src/components/icons/` 导入或内联 SVG
- [ ] **TypeScript 规范** → `<script setup lang="ts">`，`defineProps` + `withDefaults`
- [ ] **状态管理** → 如需全局状态，使用 `*Provider.vue` + `use*()` 组合式函数
- [ ] **代码质量** → 无未使用变量，Prettier 格式化，ESLint 检查通过

## 12. API 集成建议

`src/api/` 目录当前为空，没有配置 HTTP 客户端。如需添加 API 层：

### 建议目录结构

```
src/
├── api/
│   ├── client.ts           # HTTP 客户端配置
│   ├── orders.ts           # 订单相关 API
│   └── users.ts            # 用户相关 API
```

### 示例实现

```ts
// src/api/client.ts
import axios from 'axios'

export const client = axios.create({
  baseURL: import.meta.env.VITE_API_URL,
  timeout: 10000,
})

// 请求拦截器
client.interceptors.request.use((config) => {
  const token = localStorage.getItem('token')
  if (token) {
    config.headers.Authorization = `Bearer ${token}`
  }
  return config
})

// src/api/orders.ts
import { client } from './client'

export const getOrders = () => client.get('/orders')
export const createOrder = (data: any) => client.post('/orders', data)
export const updateOrder = (id: string, data: any) => client.put(`/orders/${id}`, data)
export const deleteOrder = (id: string) => client.delete(`/orders/${id}`)
```

### 环境变量

在 `.env` 文件中配置 API 地址：

```
VITE_API_URL=http://localhost:3000/api
```
